.. \+\_\+usage\+:\hypertarget{md_docs_usage_autotoc_md0}{}\doxysection{Main function}\label{md_docs_usage_autotoc_md0}
The main function a user will be using is {\ttfamily \mbox{\hyperlink{libhankel_8h_ac61e4e7e81a12c1006283542141730e8}{hankel\+\_\+transform()}}}. Please click on the name to find out what parameters the user receives as well as their types. The parameter {\ttfamily strategy\+\_\+name} essentially allows the user the select one of the 14 Hankel transform strategies available. This is further explained under \char`\"{}\+Note\char`\"{} at {\ttfamily \mbox{\hyperlink{libhankel_8h_ac61e4e7e81a12c1006283542141730e8}{hankel\+\_\+transform()}}}.

Among the input parameters to the function, there is a C struct called {\ttfamily \mbox{\hyperlink{structstrategy__params}{strategy\+\_\+params}}}. This struct needs to contain different parameters depending on {\ttfamily strategy\+\_\+name}. Clicking on {\ttfamily \mbox{\hyperlink{structstrategy__params}{strategy\+\_\+params}}} will give an idea of all possible fields this struct can contain. Each strategy only uses some of these, and it\textquotesingle{}s important that {\ttfamily \mbox{\hyperlink{structstrategy__params}{strategy\+\_\+params}}} contains the right parameters required by the selected strategy. If it doesn\textquotesingle{}t, an error will be thrown.

Here is an explanation of each parameter\+:
\begin{DoxyItemize}
\item {\ttfamily n\+\_\+eval}\+: integer indicating number of function evaluations
\item {\ttfamily eps\+\_\+rel}\+: relative error allowed e.\+g. 1e-\/9
\item {\ttfamily f\+\_\+max}\+: float indicating starting guess for max in form factor
\end{DoxyItemize}

Please note, for DHT strategies (that is, DHT\+\_\+6 to DHT\+\_\+11) feel free to supply an empty struct for {\ttfamily \mbox{\hyperlink{structstrategy__params}{strategy\+\_\+params}}} . Any field within it will be ignored in these strategies as they do not require such additional parameters. ~\newline
